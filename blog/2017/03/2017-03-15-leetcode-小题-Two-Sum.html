leetcode 小题 Two Sum

发表于 2017-03-15 | 分类于 算法

引言

今天在装vs的build的时候闲着没事做，打开leetcode刷了一个简单的小题，然后发现这个题套路还挺深的，并没有我起初想的那么简单，解法还有多种。

解题分析

首先我把题目放在下面

Given an array of integers, return indices of the two numbers such that they add up to a specific target.

You may assume that each input would have exactly one solution, and you may not use the same element twice.

Example:

Given nums = [2, 7, 11, 15], target = 9,

Because nums[0] + nums[1] = 2 + 7 = 9,
return [0, 1].

没有用什么有趣的解法，仅仅是套了一个二重循环，做了一个相等判断，相信任何学过c的人都能轻松地想到这种解法，同时这也是一个很普通的时间复杂度为O(n^2)的解法，然后我又在答案页上看到了这种解法：

这个解法在第一次遍历数组先将所有元素和它的下标作为key-value对存入Hashmap中，第二次遍历数组时根据目标和与当前元素之差，在Hashmap中找相应的差值。如果存在该差值，说明存在两个数之和是目标和。此时记录下当前数组元素下标并拿出Hashmap中数组元素下标即可。Hashmap获取元素的时间复杂度是O(1)，所以总的时间复杂度仍不超过O(n)。

这个，怎么说呢，我表示我完全想不到这种方法啊我的哥，本来一个easy的题，还有这么多套路，我还是太年轻了。
