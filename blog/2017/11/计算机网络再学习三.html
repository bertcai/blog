计算机网络再学习三

发表于 2017-11-29 | 分类于 计算机网络

引子

离上一期计算机网络再学习已经有不止几天了，现在继续新的计算机网络再学习之旅(✺ω✺)。同时，前面两篇把整个计算机网络的体系大概过了一遍，从下面开始就是详细展开了，所用的书是自顶向下的叙述方式，所以现在就开始应用层的学习。

应用层协议原理

网络应用程序体系结构

  • CS(clien-server)结构
    • 有一个总是打开的主机称为服务器，他服务来自许多称为客户的主机的请求
    • 客户相互之间不通信
    • 服务器具有固定的、周知的地址
    • 常见的该体系应用FTP、电子邮件等
  • P2P结构
    • 对服务器有最小的依赖
    • 应用程序在间断的主机对之间使用直接通信，这些主机对即称为对等方
    • 主要用于下载、文件共享与视屏聊天等领域

进程通信

  • 客户与服务器进程

  • 进程与计算机网络的接口

    • 即socket
  • 进程寻址
    • 主机的地址（IP）
    • 定义在目的主机中的接受进程的标识符（PORT）

可供应用程序使用的运输服务

运输层协议负责使报文进入接受端进程的socket，它需要提供四点服务

  • 可靠数据传输
    • 某些容忍丢失的应用可能会接受部分数据丢失，如交谈式音视频
  • 吞吐量
    • 带宽敏感性应用对吞吐量要求很高
    • 弹性应用，如电子邮件、web传输等，能根据情况调整。
  • 定时
    • 通话和在线游戏
  • 安全性

因特网提供的运输服务

在因特网中有两个重要的运输层协议，它们是TCP和UDP

  • TCP服务
    • 面向连接服务
      • 在应用层报文数据开始流动之前，TCP让客户和服务互相交换传输层控制信息，即握手过程。握手过程结束后，一个TCP连接就建立完成了，这个连接是全双工，即连接双方可以同时发送和接受信息。
    • 可靠数据传送服务
      • 通信进程能够依靠TCP，无差错、按适当顺序交付所有发送的数据，即它保证了数据的可靠性。
    • TCP提供了拥塞控制机制

  • UDP服务

    • UDP是一种不提供不必要服务的轻量级传输协议，它提供最小服务。
    • UDP是无连接的，两个进程通信间没有握手过程。
    • UDP提供一种不可靠数据传输服务。
    • UDP没有包括拥塞控制机制。

  • 运输层协议没有提供的服务

    • 吞吐量
    • 定时（时延）

应用层协议

应用层协议定义了运行在不同端系统上的应用程序如何相互传递报文，特别是应用层协议定义了：

  • 交换的报文类型，请求报文和响应报文
  • 各种报文类型的语法
  • 字段的语义
  • 一个进程何时以及如何发送报文，对报文进行响应的规则

Web和HTTP

HTTP概况

Web的应用层协议是超文本传输协议（HTTP），HTTP由两个程序实现：

  • 客户程序
  • 服务器程序

这两个程序运行在不同的端系统中，通过交换HTTP报文进行会话。

HTTP依赖于TCP的支撑，客户端向服务端发送一个HTTP请求，然后服务端回送一个HTTP响应。一个重要的部分，服务器向客户发送被请求的文件，但是并不存储客户的信息，所以即使客户在短短几秒之间进行两次请求同一个对象，服务器并不会因为刚刚为客户提供了该对象就不再做反应，而是重发对象。这就是HTTP无状态协议的体现部分。

但是

  • 通过增加cookie和session机制，现在的网络请求其实是有状态的
  • 在没有状态的http协议下，服务器也一定会保留你每次网络请求对数据的修改，但这跟保留每次访问的数据是不一样的，保留的只是会话产生的结果，而没有保留会话

非连续连接和持续连接

  • 非连续连接
    • 即每个请求响应对是进过一个单独的TCP连接发送
    • 缺点：需要建立和维护许多TCP连接，造成资源浪费
    • 缺点二：每一个对象经受两倍RTT的交付时延。
  • 持续连接
    • 所有请求即=及响应经相同的TCP连接发送

HTTP报文格式

  • 请求报文
    • 请求行
    • 首部行
    • 空行
    • 实体
  • 响应报文
    • 状态行
      • 状态码
      • 2xx OK
      • 3xx 对象转移
      • 4xx 请求不能被服务理解
      • 5xx 服务器报错
    • 首部行
    • 实体

cookie

  • 响应报文首部
  • 请求报文首部
  • 用户端浏览器保留管理一个cookie文件
  • web服务器后端数据库

cookie标记了用户，当用户再次访问目标网站时，服务器通过数据库中的cookie找出对应的表项加载给用户，但是cookie也带来了一些安全问题，用户隐私等。

web缓存器

也称代理服务器，当客户端发送请求时，浏览器先建立一个到web缓存器的TCP连接，若缓存器类有对象副本，则直接返回该副本对象，若没有，则继续向初始服务器请求对象。

  • 好处
    • 大大减少对客户请求的响应时间
    • 大大减低因特网上的web流量，改善应用性能

文件传输协议：FTP

FTP需要两条TCP连接

  • 一个是控制连接，用于传输两主机之间的控制信息，如用户标识、口令等。
  • 一个是数据连接，用于实际文件传输。

FTP常见命令

  • USER username： 向服务器传送用户标识
  • PASS password：向服务器发送用户口令
  • LIST： 列出文件
  • RETR filename：用于从远程主机当前目录检索文件
  • STOR filename: 用于在远程主机当前目录上存放文件
